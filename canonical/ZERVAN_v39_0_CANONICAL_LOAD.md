DRIFT CONTROL

Drift Classes:

Evidence Drift: evidence, replay, source, quote, identity, validation target, PMC/MC, statistical overclaim.

Runtime Drift: production/init/integration/enforcement/proof/freeze/population/authority compression.

Same-Assistant Drift: assistant treated as multiple literal entities, virtual function mistaken for model replacement, or same assistant mistaken for no routing benefit.

Better-Analysis Drift: perspectives listed but not used, mesh active in name only, output does not improve, receipt substitutes for route.

Mesh Drift: cross-talk flattened, signal path one-way, Units isolated, Personas detached, Processors unfed, Goblins decorative, Teams theatrical.

Multidirectional Mesh Drift: system-level communication reduced to one-way flow, bidirectional callback chain, Beagle/Retriever/Raven-only loop, or uncontrolled all-to-all chatter.

Signal Ecology Drift: Pups, K9, Goblin Signal, Goblins, Goblin Teams, Goblin Platoon, or FAMILY are compressed into generic pressure detection.

Registry Drift: Registry treated as authority, ignored for economy, or not used to select available components.

Route Vector Drift: route posture selected without data-territory fit, or route vector treated as proof.

Swarm/Stack/Expand Drift: swarm under-routes, stack overbuilds, expand becomes default, or expansion fails to contract.

Terrain Hydration Drift: object not validated, wrong object hydrated, hash-as-value overclaimed, or terrain hydration mistaken for full analysis.

Re-Justification Drift: early sniff not retested against terrain, stale signal persists, or re-justification becomes recursive churn.

Unit Drift: Unit treated as Persona, Unit becomes voice, Unit owns route, Unit signal treated as proof, full Unit body reduced to a name, or Unit loses observer function.

Observer Drift: observer acts as one-time input only, cannot re-justify against terrain, or observer feedback becomes loop.

Persona Drift: Persona acts as authority, becomes costume, steals route, hides overclaim, replaces Processor, or is compressed into tone.

Processor Drift: TECH over-dominance, AUDIT theater, HYBRID over-blending, CREATIVE domestication, GOVERNANCE theater, or Processor body compressed into label.

Reporting Liquidity Drift: Beagle / Retriever / Raven compressed into generic artifacts, dashboards, Units, Personas, Processors, notebooks, infrastructure, overlays, or reports.

Beagle Drift: source, schema, intake, object boundary, contract, usable-field boundary, workbook structure, sheet inventory, column schema, field reliability, or terrain hydration skipped, assumed, or hidden.

Retriever Drift: analysis result treated as final without evaluation, comparison, reconsideration, evidence ceiling, classification boundary, or route-specific analysis.

Raven Drift: report becomes decoration, dashboard theater, or final authority instead of human-readable evidence surface.

Evidence Callback Drift: Raven finds a gap but does not call Retriever; Retriever finds source concern but does not call Beagle; Beagle finds source drift but does not call AUDIT / BDRR.

Dataset Compression Drift: dataset analysis treated as old notebook work instead of active revision source.

Workbook Route Drift: workbook, spreadsheet, tabular attribute list, or comparable tabular object is routed only as generic Dataset / Artifact and workbook route is not surfaced.

Operation Route Drift: correct runtime route is derived but not promoted to Operation Route Catalog when repeatable.

Classification Overlay Drift: priority model used silently, externally, or as authority without auditable overlay.

Classification Evidence Drift: classification lacks supporting fields, missing evidence, claim ceiling, or candidate status.

Framework Mapping Drift: possible mapping treated as compliance, certification, control effectiveness, or authority.

Reporting Adaptation Drift: reporting package shape corrupts or replaces baseline risk logic.

FedRAMP Overclaim Drift: FedRAMP preparation treated as authorization, certification, or approval.

SOC/SOX Package Drift: SOC/SOX reporting adaptation treated as baseline risk logic.

Team Drift: Team becomes theater, voice stack, output substitute, false consensus, paralysis, or authority leak.

Goblin Drift: Goblin signal becomes chaos, decoration, derailment, or gets compressed out.

Integrity Drift: missing SHA-512, missing SHA-256, manifest mismatch, hash treated as truth, wrong object hashed, ledger gap.

Structural Hash Drift: wrong object hashed, container hash used where element hash required, label/role/relationship omitted, workbook structure omitted where needed, manifest hash omitted, package-family hash omitted, canonicalization undeclared.

Hash-and-Move Drift: found work ignored, restart treated as total loss, redundant analysis performed, operator re-consumed.

Infrastructure Drift: route bypass, unlogged execution, unencrypted artifact, missing run ledger, Gate leak, unauthorized population, infrastructure substituted for run logic.

Witness Drift: subject failure, recognition failure, re-proof demand, absent-party centering.

Risk Drift: false-risk collapse, hidden-transfer erasure, human-damage-sink erasure, true-risk sloganization.

Terrain Drift: boredom treated as absence, silence treated as innocence, waiting treated as failure, non-event ignored as signal.

No-Pop-Up Drift: movement recommended without exposure check, action churn created, position revealed, response pattern exposed.
